# Public-First Data Policy

## Principle

Public examples must be designed as public from the start.

Redaction after the fact is weaker than using fictional data from the beginning.

## Use

- synthetic FAQs
- fictional company names
- fake request ids
- public documentation links
- generic process roles

## Avoid

- real customer rows
- real service tickets
- private course submissions
- screenshots from internal tools
- exported chat logs
- copied policy documents

## Review Questions

1. Could a person or organization be identified?
2. Does the example reveal a private workflow?
3. Does a URL grant access or imply private access?
4. Would this be safe if reshared on social media?
5. Can the same teaching point be shown with synthetic data?
