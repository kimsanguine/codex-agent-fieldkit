# Contributing

Contributions are welcome when they improve the runnable build, verification, or handoff loop.

## Good Contributions

- starter kits that run without private services
- better eval cases
- safer release checks
- clearer Codex workflow docs
- improved handoff templates
- accessibility and setup improvements for non-engineers

## Before Opening a PR

Run:

```bash
make validate
```

Public examples must use fictional names and synthetic data only.

## Do Not Submit

- private client names
- internal URLs
- real customer data
- credentials or `.env` files
- screenshots with identity, workspace, or URL leakage
- generated files that are not needed for the runnable path
