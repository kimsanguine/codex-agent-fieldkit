# Agent Fieldkit Scorecard

Score each area from 0 to 10.

## 1. Open-Source Growth

9+ means:

- README explains the repo in 10 seconds.
- First command path works.
- The repo is not a generic prompt list.
- It has a shareable story and concrete artifact.
- It gives another repo or team a reason to link back.

## 2. Codex Education Design

9+ means:

- Codex is central, not decorative.
- The learner opens the right folder.
- `AGENTS.md` is used as project memory.
- Inspection-before-edit is taught.
- The run/debug/verify loop is explicit.
- Handoff is part of the workflow.

## 3. Governance And Safety

9+ means:

- All examples are synthetic or public.
- Private-term and secret scans exist.
- Release checklist exists.
- Handoff avoids account/key transfer.
- Limitations are explicit.

## 4. PM/CPO Usefulness

9+ means:

- The repo shows idea to delivery.
- It includes PRD, architecture, progress, validation, and handoff artifacts.
- It is understandable without deep engineering background.
- It makes verification visible.

## 5. Corporate L&D / AX Usefulness

9+ means:

- Facilitators can explain prerequisites and stop conditions.
- The default path avoids private data.
- Rubric and validation evidence are present.
- It can be shared with an internal AI task force for review.

## 6. Developer Credibility

9+ means:

- `make demo`, `make test`, `make eval`, and `make validate` work.
- CI runs the same checks.
- The code is small and inspectable.
- Tests cover behavior and fallback.
- The repo includes a repo-scoped Codex skill or equivalent workflow asset.

## Public Release Threshold

Average score must be at least 9.0, with no category below 8.5.
